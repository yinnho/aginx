# Aginx 开发计划

## 阶段一：核心安全问题（P0）

### 1.1 设备身份与硬件指纹

**目标**: 实现 aginx 设备的唯一标识，防止假冒

**任务**:
- [ ] 1.1.1 实现硬件指纹生成模块
  - 获取 MAC 地址
  - 获取硬盘序列号
  - 获取主板 UUID
  - 生成组合指纹 hash
- [ ] 1.1.2 修改注册流程
  - 首次启动发送硬件指纹到远程
  - 远程返回/分配 aginx_id
  - 本地保存 aginx_id
- [ ] 1.1.3 修改重连流程
  - 重置后重新计算硬件指纹
  - 远程根据指纹返回原有 aginx_id
- [ ] 1.1.4 远程服务配合修改
  - 存储 {硬件指纹: aginx_id} 映射
  - 验证指纹匹配

**验收标准**:
- 重置 aginx 后，aginx_id 保持不变
- 不同电脑无法使用相同 aginx_id

**预估工作量**: 2天

---

### 1.2 配对码验证漏洞修复

**目标**: 配对码必须正确才能绑定

**任务**:
- [ ] 1.2.1 修改 PairCodeData 结构，保存实际 code
- [ ] 1.2.2 修改 save_pair_code，保存 code 值
- [ ] 1.2.3 修改 load_pair_code，验证 code 匹配
- [ ] 1.2.4 添加测试用例

**验收标准**:
- 只有正确的配对码才能通过验证
- 错误配对码返回失败

**预估工作量**: 0.5天

### 1.3 App 绑定独占模式

**目标**: 一个 aginx 只能被一个 App 绑定

**任务**:
- [ ] 1.3.1 修改 DeviceInfo 结构，改为单一绑定
- [ ] 1.3.2 修改 bindings.json 格式
- [ ] 1.3.3 绑定前检查是否已绑定
- [ ] 1.3.4 已绑定时返回错误提示（需先解绑）
- [ ] 1.3.5 添加解绑 API
- [ ] 1.3.6 添加测试用例

**验收标准**:

**预估工作量**: 1天

3. **兼容性**: 修改 bindings.json 格式可能影响已绑定设备
